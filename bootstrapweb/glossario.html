Glossario

Un processo è un'istanza di un programma in esecuzione, che include codice, dati e risorse del sistema operativo.

TCP (Transmission Control Protocol) è un protocollo di comunicazione affidabile che garantisce l'ordine e l'integrità dei dati tra due host in una rete.

UDP (User Datagram Protocol) è un protocollo di comunicazione che trasmette dati senza stabilire una connessione, con minore affidabilità ma maggiore velocità rispetto a TCP.

Il client è un'applicazione o un dispositivo che richiede un servizio da un server in una rete, inviando richieste e ricevendo risposte.

Il server è un sistema o applicazione che fornisce risorse o servizi a un client, rispondendo alle sue richieste in una rete.

Un protocollo è un insieme di regole che definiscono come i dati vengono trasmessi e ricevuti tra i dispositivi in una rete.

Unix è un sistema operativo multitasking e multiutente, noto per la sua stabilità, sicurezza e la struttura di file system gerarchica, ampiamente utilizzato in ambienti server e di sviluppo.

Un indirizzo IP è un identificatore numerico unico assegnato a ogni dispositivo connesso a una rete, utilizzato per localizzare e comunicare con altri dispositivi su Internet o una rete locale.

Una porta è un numero che identifica un'applicazione o un servizio specifico su un dispositivo di rete, utilizzato per indirizzare correttamente i dati tra client e server.

Una rete locale (LAN) collega dispositivi vicini, come in un ufficio o abitazione, per condividere risorse e dati in modo rapido e sicuro.

Un database è un sistema organizzato per archiviare, gestire e recuperare dati in modo efficiente tramite software specific.

I segmenti sono blocchi di dati divisi per il trasporto in rete, specialmente nel contesto del protocollo TCP, per garantire una trasmissione efficiente e ordinata.

I datagrammi sono unità di dati indipendenti utilizzate principalmente nei protocolli senza connessione come UDP, che vengono inviati senza garantire ordine o affidabilità nella consegna.
